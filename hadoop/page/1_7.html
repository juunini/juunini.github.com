이번 글에서는 HDFS 명령어에 대해 알아보겠습니다.

HDFS명령어는 대부분 리눅스 명령어와 유사한점이 많습니다.
사용법은 터미널에서 하둡 폴더 내에서
./bin/hadoop fs -명령어 [옵션]
이런식으로 사용하게 됩니다.

./bin/hadoop fs -ls [디렉터리 or 파일]
해당 파일이나 디렉터리 내의 파일 정보를 봅니다.

./bin/hadoop fs -lsr [디렉터리]
해당 디렉터리 내의 하위 디렉터리들의 정보까지 다 봅니다.

./bin/hadoop fs -du [디렉터리 or 파일]
해당 파일이나 디렉터리의 사용량을 확인합니다.

./bin/hadoop fs -dus [디렉터리]
해당 디렉터리의 사용량 합계를 봅니다.

./bin/hadoop fs -cat [파일]
파일의 내용을 읽어옵니다.

./bin/hadoop fs -text [파일]
cat 명령어는 압축된 파일은 못읽어오지만 text 명령어는 압축된 파일도 읽어옵니다.

./bin/hadoop fs -mkdir [폴더이름]
폴더를 만듭니다.
./bin/hadoop fs -put [디렉터리 or 파일] [디렉터리 or 파일]
$HADOOP_HOME에 있는 파일 또는 폴더를 hadoop-data 폴더로 복사합니다.

./bin/hadoop fs -get [디렉터리 or 파일] [디렉터리 or 파일]
hadoop-data 폴더에 있는 파일 또는 폴더를 $HADOOP_HOME으로 복사합니다.

./bin/hadoop fs -getmerge [디렉터리] [생성할 파일 이름]
디렉터리 내의 모든 파일의 내용을 합쳐서 하나의 파일로 복사한다.

./bin/hadoop fs -cp [출발지 디렉터리, 파일] [목적지 디렉터리, 파일]
출발지 디렉터리나 파일을 목적지로 복사합니다.

./bin/hadoop fs -mv [출발지 디렉터리, 파일] [목적지 디렉터리, 파일]
출발지 디렉터리나 파일을 목적지로 이동합니다.

./bin/hadoop fs -rm [파일]
파일을 삭제합니다.
./bin/hadoop fs -rmr [디렉터리]
디렉터리를 삭제합니다.


자주 사용하는 것 위주로 적었습니다. 이것만 알아도 특별히 막히거나 하진 않을 것 입니다.
